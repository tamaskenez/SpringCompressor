# Compressor dB transfer curve, soft knee formula

The gradual, even increase of the ratio is exponential R * exp(d). The derivative on transitional region of the knee must increase in an exp(dx) manner.
If the threshold is at x0, the linear region of the curve starts at x1 > x0, the transitional region is x0..x1, then the transitional curve f(x):

f(x0) = x0
f'(x0) = 1
f'(x1) = 1/R

f = A * exp(B * (x - x0)) + C

which results in:

w = x1 - x0;
B = log(1/R) / (x1 - x0);
A = 1 / B;
C = x0 - A;

This results in a most gradual transition from R = 1 to R.

## Summary report

I've been experimenting with audio dynamic range compressor algorithms. The solution I'm looking for is a compressor which performs its duty, to compress the dynamic range of the input as cleanly and transparently as possible, with the least amount of coloring.
I'd like to combine basic, simple building blocks, without complicated, special solutions to handle corner cases and without emulating existing hardware or software solutions.
I'd like to believe that there are objective, measurable goals which characterize the ideal compression I'm aiming for, and these are:

1. Accurately following the level transfer curve set by the user (threshold, ratio)
2. Keeping or suppressing the transients as dictated by the attack control
3. Keeping the harmonic distortion as low as possible

Let me elaborate on these:

1. Accurately following the level transfer curve set by the user (threshold, ratio)

By accurately following I mean that for a steady sine wave the input and output RMS values must follow the formula `out_db = (in_db - threshold_db) / ratio + threshold_db`
To achieve this we need an accurate RMS measurement. To be as musical sounding and transparent as possible the idea is to rely on RMS envelope instead of peak envelope.

2. Keeping or suppressing the transients as dictated by the attack control

For lower settings of the attack control we want to attenuate transients by applying the transfer curve sooner when the input signal's amplitude is increasing quickly.
For higher settings of the attack control we want to let transients through and start applying the transfer curve later, gradually.

Unlike traditional compressors we are not primarily interested in providing a release control. The reason is that the user wants to control the smearing of transients, they don't actually need a release control. The release control and the asymmetric setup of the gain control logic (short attach/long release) is there just as a workaround, to provide shorter attack times without excessive audio frequency content appearing on the gain reduction signal which translates into harmonic distortion on the output. The release control just a fix to compensate a side-effect of the compressor and does not express intent of the user.

So, ideally, in my compressor I'd like to provide an attack control and a harmonic distortion suppression control which automatically selects the lowest level of release which provides the selected level of harmonic distortion suppression.

3. Keeping the harmonic distortion as low as possible

The harmonic distortion arises from audio frequencies appearing in the gain reduction signal which we use to modulate the input signal with.
We have contradicting requirements: we want the gain reduction signal as smooth as possible but we want to follow the input signal's envelope as closely as required by the attack control. The difficulty is to separate the input signal's envelope (RMS) from these two other effects which are not part of the envelope:

- The natural vibration of the sound wave at its frequency: The real (non-imaginary) audio signal alternates between its negative and positive amplitudes. Since the RMS detector works by low-pass filtering the squared input signal, the natural vibration of the wave shows up as double frequency sine wave on top of the signal's power as DC component. This double frequency component needs to be filtered out by the low-pass filter in order to not be mistaken for the envelope.
- Interference between different frequency components: Two sine waves with frequencies f1 and f2 produce and amplitude modulation with difference frequency.

So, the difficulty is that the quickly raising amplitude of a transient (which is part of the envelope) moves as fast or faster than the latter two non-envelope related effects.
I had two ideas to overcome this problem:

1. Split the input signal with a filter bank and detect signal power separately for different bands. Use low-pass filters with time constants tuned to the bands frequencies. This way we can use slow low-pass filters for bass frequencies and fast ones for higher frequencies
2. Use Hilbert transform to compute the analytical signal and measure RMS there. For a pure sine wave the analytical signal's amplitude is constant, it is not alternating, even with zero filtering the RMS reading is perfect.

I made some experiments and this is what I've found:

The filter bank does not solve the interference problem. Two signal components at 1000 Hz and 1100 Hz will create a ripple in the RMS signal at 100 Hz which will not be suppressed by the low-pass filter tuned to 1000 Hz signals.
The filter bank does solve the problem of suppressing the double frequency ripple appearing in the RMS signal. However, that is already solved by the Hilbert/analytical method.

That's why my conclusion is that we probably don't need a filter bank. Unless we can afford having quicker low-pass filters for RMS detection for higher frequencies because we think those regions are less probably to contain high-energy, low-frequency interferences. I'm not sure if that's true.

My current design idea is the following architecture:

- Measure the RMS of the input signal by computing the analytical signal with Hilbert transform, taking the norm of the complex valued samples (squared absolute value) and feeding it into a low-pass filter which is tuned according to the attack control. It should keep the transients which we need to preserve for the current setting.
- Then, apply a second low-pass filtering. This time it's asymmetric: on raising edges immediately take the input value, on falling edges apply the amount of low-pass filtering which required for the harmonic distortion target.
- Then, apply the transfer curve.
- Then, apply a third, light, low-pass filtering to smooth out any abrupt changes the transfer curve might have introduced.

Please let me know how you evaluate my design and ideas and provide insights if you have any.
Also I'd be interested to know if there's anything I can learn from historical, existing compressor architectures and incorporate it into my design.


## Design Guidelines

- Correct by design, as opposed to fix problems by tuning
- Verified by objective measurements
- Small number of "moving parts", simple architecture
- No analog emulation - let's enjoy the feature where digital is better
- Maximum effort to avoid digital artifacts
- The less controls the better

## Concrete, compressor-specific design decisions

- High-quality, transparent oversampling
- Minimize harmonic distortion, inherent and sometimes preferred in traditional compressor architectures
- Attack times controlled by peak detection is a compromise to allow fast reaction times without
  introducing excessive harmonic distortion. This compressor uses a cleaner design, a frequency dependent
  RMS detection which allows very short attack/release times without harmonic distortion.
- As smooth as possible (exponential) soft knee

### State variable TPT state update after coefficient change.

The original equations, merged together:

    s1n = 2 * g * h * xi + (1 - 2 * g * h * (R2 + g)) * s1p - 2 * g * h * s2p;
    yLP = g^2 * h * xi + (g - g^2 * h * (R2 + g)) * s1p + (1 - g^2 * h) * s2p;
    s2n = 2 * g^2 * h * xi + (2 * g - 2 * g^2 * h * (R2 + g)) * s1p + (1 - 2 * g^2 * h) * s2p;
    y(i) = yLP;

Problem:

- s1 tracks the slope of the input, but the encoding depends on the g coefficient.
- s2 tracks the DC of the input but the encoding depends on s1 and R2 (or g and R2).

State update methods assume a steady ramp (x[i] = C + D * i). In this case the state variables converge to this:

    s1 = D / (2 * g);
    s2[i] = x[i] + (g - R2) * D / (2 * g) = x[i] + (g - R2) * s1

In this ase

We try to make the transition as smooth as possible.

Method 1: not updating the state variable.

Result after changing the coefficients:
    s1 gradually changes to its new value which encodes the same slope.
    yLP (and s2) discontinuity for 2 samples.

Method 2: updating the state variable to its new steady values:

Result after changing the coefficients:
    s1 and s2 stays steady
    yLP (and s2) discontinuity for 1 sample
This method is the best for BP and HP outputs. For LP output, if the application allows, we can add a new offset
to yLP after each change. This might not be suitable when the filter needs to track DC.

Method 3: s1 unchanged, update s2 such that xi would have resulted in the same yLP.
Result: Compared to not updating:
    s1 similar
    yLP (and s2), instead of the 2 sample jump, the first sample continues the previous ramp and a double jump on the second
    So, it's worse than not updating.

Method 4: s1 to steady, s2 stays the same.
Result:
    s1 quickly changes to another, intermediate value (between the old and new steady) then gradually changes
        to the new steady.
    yLP (and s2) no serious discontinuity but there is a back-and-forth transient before it settles on steady.
    best for LP output.

Equations for updates:

s1 = D / (2 * g);
s2n = yLP + D / 2;
s2n = xi + (g - R2) * D / (2 * g);

yLP = s2n - g * s1;
D = 2 * g * s1;
xi = s2 - (g - R2) * s1;

## Lossy integrator

low-pass butterworth filter:

[b, a] = butter(1, fc);
b(1) == b(2), let's call it b.
a(1) = 1, let's a = a(2).

Y/X = (b + b * z^-1) /(1 + a * z^-1)
Y (1 + a * z^-1) = X (b + b * z^-1)
y[n] + a * y[n-1] = b * x[n] + b * x[n-1]
y[n] = b * x[n] + b * x[n-1] - a * y[n-1]

How much does y grows:

    dy = y[n] - y[n-1] = b * x[n] + b * x[n-1] - a * y[n-1] - y[n-1]
    = b * (x[n] + x[n-1]) - (a + 1) * y[n-1]
    = 2 * b * (x[n] + x[n-1]) / 2 - (a + 1) * y[n-1]

That means, that aside from the lossy term `(a + 1) * y[n-1]` and taking the sample time as unity, the
filter integrates 2 * b * x.

We need to estimate when the filter output goes to zero.

    y[n] + dy/dt * DT = 0

Since

    y = integral(2 * b * x, dt)
    dy/dt = 2 * b * x

    y[n] + 2 * b * x[n] * DT = 0
    DT = -y[n] / (2 * b * x[n])

Test:

[b, a] = butter(1, 0.001);
fc = 0.01;
ns = [0:1000]';
x = sin(2 * pi * ns * fc);
y = filter(b, a, x);
DT = -y ./ (2 * b(1) * x);
DT(DT < 0) = nan;

R = 600:800;
subplot(2,1,1);
plot(ns, x, ns, y), grid, legend('x', 'y'), axis([R(1) R(end) -1 1]);
subplot(2,1,2); plot(ns, ns + DT), grid, axis([R(1) R(end), R(1), R(end)]);


Tracking the average value of the current peak/through:

`slow` and `fast` are LPF(input.^2) with lower and higher cutoff frequencies.
fms = fast - slow, the difference signal
i_fms = LPF(fms), where LPF is a lossy integrator (for know I'm using the slow cutoff freq for it)
i0 is the position of the last sign change of i_fms
mean_i_fms = i_fms(i) / (i - y0)  is the average fms since the last sign change
When abs(mean_i_fms) is growing it indicates that the increase or decrease of the `fast` signal is a peak or
a through compared to the `slow` signal, as opposed to some ripple around the `slow` signal.

How does the rate of change of abs(mean_i_fms) look like?

    mean_i_fms(i) = i_fms(i) / (i - i0)
    d mean_i_fms(i) = i_fms(i) / (i - i0) / di =

Because (f/g)' = (f'g - fg') / g^2:

    = (d i_fms(i) / di * (i - i0) - i_fms(i) * d (i - i0) / di) / (i - i0)^2
    = (fms(i) * (i - i0) - i_fms(I)) / (i - i0)^2
    = fms(i) / (i - i0) - i_fms(i) / (i - i0) ^2
    = (fms(i) - mean_i_fms(i)) / (i - i0)

Making it scale invariant
    = (fms(i) - mean_i_fms(i)) / slow(i) / (i - i0)
